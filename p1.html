Project #1: Conditional Design

Description

I translated human language instructions depicting a shape, into p5 code. The exersise compares how a human might describe things and how computer describes things.

Shae

Instructions:
(light green 3 - G34, H34, K34, L34, G35, H35, K35, L35, N35, O35, H36, I36, K36, L36, N36, O36, I37, J37, K37, L37, M37, N37)
(light red 3 - I38, K38, L38, M38, N38, H39, I39, J39, L39, M39, N39, O39, H40, J40, L40, M40, N40, H41, I41, J41, K41, L41, N41, O41, H42, I42, J42, K42, L42, M42, N42, O42, I43, J43, L43, M43, N43, I44, J44, K44, L44, M44, N44, J45, K45, L45, M45, J46, K46, M46, K47, L47)
(black - K39, I40, O40, M41, K43, L46)

Reflection

Human language and computer code both depend on context. But you can’t always assume the other party has the same context you do. Creating definitions upfront helps smooth the process as you go between languages.

A code is dependent on a key. Once you’ve established it you can plug things in and the proper outputs will appear. But human language is more open and has a wider array of meanings, that might change according to situation or person.

Human language and computer code both have patterns and syntax. Learning how these fit together can help us build things or expand/glitch/open things.
